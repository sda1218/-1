### 更新功能状态

随着功能推进，逐步填充文档链接并更新状态：

1. **需求分析完成**：填写需求文档链接，状态改为 `🟡 开发中`
2. **设计完成**：填写设计文档链接
3. **测试开始**：状态改为 `🟠 测试中`
4. **文档同步完成**：填写 API 文档链接
5. **CI 通过**：状态改为 `🟢 已交付`，更新最后更新日期

### 文档路径格式

文档链接使用相对路径，格式为：

- 需求文档：`01-product-specs/{功能英文名}-spec.md`
- 设计文档：`02-design-docs/{功能英文名}-design.md`
- API 文档：`04-api-docs/{模块英文名}-api.md`

如功能尚无对应文档，填写 `—`（破折号）。

## 功能详细记录

### 无人机信息管理系统

**功能概述**：基于 Spring Boot 2.2.x + MyBatis 3.5.x 的无人机信息管理系统，采用标准四层架构，支持 SQLite/MySQL 数据库切换，使用 Apache Shiro 进行安全认证和授权。

**实现的功能点**：

1. **无人机信息录入（创建）**
   - 支持完整的无人机信息录入（序列号、型号、制造商、购买日期、状态、飞行参数、重量、备注）
   - 序列号唯一性校验
   - 参数校验（必填字段、格式验证、数值范围验证）
   - 自动生成唯一标识符和时间戳

2. **无人机信息查询**
   - 根据 ID 查询单条记录
   - 分页查询列表（支持自定义页码和每页数量）
   - 条件查询（型号、制造商、状态）
   - 模糊查询支持

3. **无人机信息修改（更新）**
   - 支持部分字段更新
   - 序列号重复校验
   - 参数合法性验证
   - 自动更新时间戳

4. **无人机信息删除**
   - 根据 ID 删除记录
   - 删除前存在性校验
   - 物理删除（非软删除）

5. **数据库切换支持**
   - 适配器模式实现 SQLite/MySQL 切换
   - 独立的 Mapper XML 文件（按数据库类型分目录）
   - 配置文件驱动的数据源选择
   - SQLite 适用于小规模部署，MySQL 适用于生产环境

6. **Apache Shiro 安全认证和授权**
   - 基于角色的访问控制（RBAC）
   - 管理员角色：创建、修改、删除权限
   - 普通用户角色：查询权限
   - 自定义 Realm 实现用户认证和授权

7. **请求拦截器（日志记录）**
   - 拦截所有 HTTP 请求
   - 记录请求详细信息（方法、路径、参数、IP、User-Agent、时间戳）
   - 记录响应状态码和处理耗时
   - 慢请求警告（>1000ms）

8. **前端页面（Thymeleaf + Bootstrap）**
   - 无人机列表页面（支持分页和条件查询）
   - 无人机详情页面
   - 无人机创建表单
   - 无人机编辑表单
   - 响应式设计（Bootstrap 3.3.7）

**技术架构**：

- **Controller 层**：`DroneController`（REST API）、`DroneViewController`（页面控制器）
- **Service 层**：`DroneService` 接口 + `DroneServiceImpl` 实现
- **Domain 层**：`Drone` 实体、`DroneDTO`、`CreateDroneRequest`、`UpdateDroneRequest`、`DroneQueryRequest`
- **Repository 层**：`DroneRepository`（MyBatis Mapper）+ 数据库适配器（`SQLiteAdapter`、`MySQLAdapter`）
- **Interceptor 层**：`RequestInterceptor`（请求日志拦截器）
- **Exception 层**：全局异常处理器 + 自定义业务异常
